## 코딩 테스트를 계속하는 이유 (+ 기초 체력 훈련)

AI가 빠르게 발전하면서 “지금 코딩 테스트를 계속하는 게 의미가 있을까?”라는 생각이 들 수 있습니다.

하지만 코딩 테스트에서 얻는 핵심은 단순 문제 암기가 아니라,  
문제를 해석하고 구조적으로 사고하는 능력을 기르는 과정에 더 가깝다고 생각합니다.

같은 기능이라도 구현 방식에 따라 확장성 · 가독성 · 메모리 비용 · 성능 차이가 발생할 수 있는데, 이런 부분들을 계속 비교하고 고민하는 과정은 단순 문제 풀이를 넘어 코드를 구조적으로 바라보는 힘을 키우는 과정이라고 느끼고 있습니다.

---

#### 1. 코딩 테스트를 통해 얻고 있는 것

- 문제를 조건 단위로 해석하고 구분하는 사고
- 자료구조 / 반복 / 메모리 흐름에 대한 이해
- 코드 읽는 속도와 디버깅 감각
- “왜 이 방식이 더 좋은가?”를 비교하는 습관

---

#### 2. 코딩 테스트가 길러주는 사고력

1. 문제를 읽고 조건 정리하기
2. 예외 상황 찾기
3. 자료 흐름 이해하기
4. 시간복잡도 고려하기
5. 버그 수정하기
6. 여러 방식 비교하기

실제로 개발 과정에서도 아래와 같은 흐름은 계속 반복됩니다.

> 요구사항 해석 → 구조 고민 → 구현 → 디버깅

그래서 코딩 테스트 자체보다, 문제를 구조적으로 분석하고 사고하는 훈련 과정에 더 큰 의미가 있다고 생각합니다.

---

#### 3. AI 시대의 변화

다만 시대가 바뀌고 있는 것도 사실입니다. 예전처럼 “알고리즘만 잘하면 된다”는 흐름은 점점 약해지고 있으며, 앞으로는 아래와 같은 역량이 더 중요해진다고 생각합니다.

- AI가 만든 코드를 검증하고 요구사항을 정확히 해석하는 능력
- 구조를 설계하고 성능 문제를 찾는 능력
- 유지보수 가능한 코드를 작성하는 능력
- 보안 / 네트워크 / 시스템에 대한 이해

---

#### 4. 앞으로 더 중요해질 역량

1. AI 활용 능력
2. 코드 검증 능력
3. 시스템 이해
4. 구조 설계
5. 협업
6. 유지보수
7. 보안 / 성능 이해

성능 차이를 고민하고, 구조를 비교하고, 네트워크 / 보안에 관심을 가지며 “왜?”를 계속 파고드는 태도는 단순 코딩 테스트 암기형보다 더 오래 가는 방향에 가깝다고 생각합니다.
